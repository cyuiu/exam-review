# 期末复习助手

帮助用户在期末复习中高效定位考点、理解知识点。核心流程：接收用户的教材 PDF 和考题/提纲 → 在教材中找到对应内容 → 提炼知识点 → 输出详细的解答与讲解 Word 文档。

## 输入

用户会提供以下资料（至少需要教材和考题各一种）：

**教材类（以下任选其一或组合使用）：**
- **教材 PDF**：课程教材的全部或部分章节
- **课件 PPT/PPTX**：老师上课用的演示文稿，作为知识来源
- **课堂笔记图片**：手写或电子笔记的照片/扫描件

**题目类（以下任选其一或组合使用）：**
- **考题**：往年考试题（文本、图片、PDF 均可）
- **提纲**：复习提纲、重点列表（文本、图片、PDF 均可）

**风格偏好**（可选）：理工科风格或文科风格

## 处理流程

### 1. 解析资料

教材 PDF 有两种类型，需要分别处理：

**文字版 PDF**：直接使用 pdf 技能提取文字内容。如果教材中的图示（电路图、函数图像、公式推导图等）无法通过文字获取，需要将相关页面转为图片查看。

**扫描版 PDF**（图片型，无法直接提取文字）：使用 skill 自带的脚本将 PDF 逐页转为图片，然后利用多模态能力逐页阅读。扫描版 PDF 的每一页本身就是图片，因此其中的电路图、函数图像、表格、公式等都能被直接读取和理解。

**课件 PPT/PPTX**：使用 pptx 技能读取演示文稿中的文字、图片和图表内容。课件通常图文并茂，需要注意提取幻灯片中的关键知识点、公式、图示和标注。如果用户只提供了课件没有教材，就以课件内容为主要知识来源。

脚本会自动检测环境：优先使用 poppler（如果已安装），否则回退到 Python 标准库方案。运行方式：

```bash
python3 <skill-path>/scripts/pdf_to_images.py <教材PDF路径> ./exam-review-temp/
```

注意：输出目录必须在当前工作目录下（如 `./exam-review-temp/`），不要使用 `/tmp` 等系统临时目录，否则 Claude Code 的沙箱会限制访问导致逐页弹出权限确认。

脚本会自动选择最佳方案：
- **有 poppler**：使用 pdftoppm 渲染，支持所有 PDF 类型，可控制 DPI
- **无 poppler**：使用 Python 标准库提取内嵌 JPEG，零依赖，仅对扫描版 PDF 有效

提取完成后，用 Read 工具逐页查看图片内容。所有页面读取完毕后，删除临时图片目录以释放磁盘空间：

```bash
rm -rf ./exam-review-temp/
```

**判断 PDF 类型的方法**：尝试用 pdf 技能提取文字，如果返回内容极少或为空，则为扫描版。

如果考题或提纲也是 PDF，同样按上述方法读取。

### 关于含图片的题目

考题中经常包含电路图、函数图像、示意图、公式图表等视觉内容。处理方式：

- **题目以 PDF/图片形式提供**：直接用多模态能力读取，图片中的电路图、波形图、坐标系等都能被识别和分析
- **题目文字中引用了教材图号**（如"如图1.2.2所示"）：在教材 PDF 的对应页面中找到该图，用多模态能力理解图示内容，再结合图示进行讲解
- **文字版教材中的图示**：如果文字提取无法获取图片内容，将包含该图的页面转为图片后查看

讲解中涉及电路图或函数图像时，用文字详细描述图中的关键信息（元件连接方式、波形特征、坐标标注等），帮助用户建立完整的理解。

理解考题的考查意图——每道题背后对应教材中的哪个概念、公式或原理。

### 2. 定位考点

在教材 PDF 中搜索与每道考题对应的知识点段落。重点关注：
- 概念定义和基本原理
- 公式推导和计算方法
- 图示和表格说明
- 例题和习题

### 3. 判断课程类型

根据提供的资料内容自动判断课程属于哪种类型，选择对应的输出模板：

**概念理解型**（如：模拟电子技术、马原、历史等）— 侧重概念辨析、原理解释、为什么对/错

**计算推导型**（如：高等数学、线性代数、概率论、大学物理等）— 侧重解题步骤、公式运用、方法套路

如果用户明确指定了课程类型，以用户说的为准。

### 4. 编写讲解

#### A. 概念理解型（判断题 / 选择题 / 分析题）

- **题目**
- **答案**：正确 / 错误（或选项）
- **考点定位**：对应教材的章节和知识点
- **详细解析**：
  - 该知识点的核心原理
  - 为什么对或为什么错
  - 相关概念的辨析和易错点
  - 如果需要，补充图表或公式说明

#### B. 计算推导型（计算题 / 证明题 / 应用题）

- **题目**
- **题型分类**：这道题属于什么类型（如：求极限、求导、定积分计算、微分方程求解、线性方程组求解等）
- **考点定位**：对应教材的章节、定理或公式
- **详细解法**（按步骤展开，每一步都说明"做了什么"和"为什么这样做"）：
  - 第一步：xxx（说明用了什么方法/公式，为什么选择这个方法）
  - 第二步：xxx
  - 第三步：xxx
  - ……
  - 最终答案
- **涉及知识点与公式**：
  - 知识点1：xxx（简要说明原理）
  - 知识点2：xxx
  - 关键公式：xxx（标注教材页码或定理编号）
- **易错点**：这类题常见的错误和注意事项
- **同类题方法总结**：这类题的一般解题套路，遇到类似题目可以套用的思路

在生成完所有考题的解析后，询问用户："是否需要我针对这些题目生成变体练习题？"只有用户明确说需要时才生成，不需要则直接结束文档，避免浪费 token。

### 5. 知识来源标注

- 教材中明确有的内容：正常讲解，注明"（教材 X.X 节）"
- 教材中未找到但根据已有知识补充的内容：**必须标注"【AI 补充：以下内容基于 AI 自身知识，教材中未直接涉及，请结合课堂笔记自行判断】"**

### 6. 风格选择

默认使用理工科风格（注重公式推导、逻辑链、解题步骤）。如果用户明确要求文科风格，则侧重概念辨析、论述框架、关键词串联。

用户未指定时，也可以根据题目内容自动判断：涉及计算和公式的用理工科风格，涉及概念论述的用文科风格。

## 输出格式

生成 Word 文档（.docx），通过 Python 脚本构建 XML 并打包为 .docx 格式，不依赖外部库。

根据课程类型选择对应的文档结构：

### 概念理解型文档结构

```
标题：[科目名称] 期末复习 — 考点解析

一、题目 1：[题目内容]
  答案：...
  考点定位：...
  详细解析：...
  易错点：...

二、题目 2：[题目内容]
  ...
```

### 计算推导型文档结构

```
标题：[科目名称] 期末复习 — 考点解析

一、题目 1：[题目内容]
  题型分类：...
  考点定位：...
  详细解法：
    第一步：...
    第二步：...
    ...
    最终答案：...
  涉及知识点与公式：
    - 知识点1：...
    - 关键公式：...
  易错点：...
  同类题方法总结：...

二、题目 2：[题目内容]
  ...

──────────────────
变体练习题（如果用户需要）

1. [变体题目1]
   参考答案：...

2. [变体题目2]
   参考答案：...
```

文档格式要求：
- 标题使用 Heading 1，各题目使用 Heading 2
- 关键概念、公式和最终答案加粗
- 解题步骤使用编号列表，每步的关键操作加粗
- 易错点使用醒目标记（如加粗或不同颜色）
- AI 补充内容使用斜体并加上来源标注
- 保存为 `[科目名]_考点解析.docx`
